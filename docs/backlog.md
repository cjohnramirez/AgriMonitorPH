# Product Backlog

| ID | User Story | Priority | Points | Acceptance Criteria |
| :--- | :--- | :--- | :--- | :--- |
| **US01** | As a **User**, I want to use the navigation bar so that I can seamlessly switch between Markets, Supply Chain, and Outlook pages. | High | 2 | - Next.js routing implemented for `/markets`, `/supply-chain`, `/outlook`.<br>- Active tab visually highlighted based on current URL. |
| **US02** | As a **Developer**, I want to create a central mock data service so that all dashboard components have consistent data to render without a database. | High | 3 | - `src/lib/data.ts` created with hardcoded JSON objects matching Figma.<br>- Data structured to be easily mapped over in components. |
| **US03** | As an **Analyst**, I want the Markets page populated with mock data so that I can view static commodity prices and macro indicators. | High | 5 | - Grid layout implemented matching Figma.<br>- Commodity cards render price, trends, and labels from mock data. |
| **US04** | As a **Logistics Manager**, I want the Supply Chain page populated with mock data so that I can view regional production and loss rates. | High | 5 | - Top palay-producing regions render with static progress bars.<br>- NFA buffer stock metrics display warning states based on mock threshold limits. |
| **US05** | As a **Farmer**, I want the Outlook page populated with mock data so that I can view static climate advisories and forecasts. | High | 5 | - ENSO and weather cards render static data.<br>- Food security indicators display current mock percentages. |
| **US06** | As a **Mobile User**, I want the grid layouts to stack responsively so that I can view the dashboard clearly on my phone. | Med | 5 | - CSS grid shifts to 1-column layout on small screens.<br>- Navigation bar remains accessible (or switches to a hamburger menu) on mobile. |
| **US07** | As a **Stakeholder**, I want visual indicators like colored trend arrows and progress bars so that I can quickly digest the data context. | Med | 3 | - Green "up" / Red "down" arrows implemented for MoM/YoY changes.<br>- Fill bars for post-harvest losses use correct color tokens. |
| **US08** | As a **Visually Impaired User**, I want the UI to have proper ARIA labels so that my screen reader can interpret the dashboard. | Low | 3 | - Image thumbnails and icons have descriptive `alt` text or `aria-hidden` attributes.<br>- Navigation links are clearly labeled. |
| **US09** | As a **Developer**, I want to ensure the components are modular so that we can easily swap mock data for an API later. | Low | 3 | - UI components (e.g., `StatCard`) are decoupled from data fetching.<br>- Data is passed as props from the top-level page components. |
| **US10** | As a **Project Manager**, I want the stateless app deployed to Vercel so that I can share a live prototype link with stakeholders. | Med | 2 | - GitHub repo linked to Vercel project.<br>- Successful production build with no linting errors. |
